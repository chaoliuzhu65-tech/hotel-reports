# dlx-competitive-analysis — 德胧定制竞品分析

> 基于通用竞品分析框架，注入酒店行业竞品维度，专为德胧集团旗下酒店（开元名都、开元曼居、百达屋等）优化。

## ⚠️ 核心战略原则：内拓外拓双轨并行

所有竞品分析动作（无论内拓/外拓场景）必须严格遵循**「存量盘活+增量增长」双目标逻辑**，该逻辑为竞品分析侧工具底层核心判断依据：
1. **存量盘活优先**：优先分析存量竞品（同区域同定位竞品、存量客户流失去向竞品）的存量激活策略
2. **增量增长同步**：在存量竞品分析基础上拓展增量竞品分析，禁止单一维度分析（仅分析增量新竞品不做存量竞品应对，或仅分析存量竞品不做增量拓展均不符合要求）
3. **外拓适配规则**：外部增量竞品分析必须同步配套存量客户防流失策略（如针对竞品增量策略的存量客户留存方案）
4. **效果评估标准**：所有竞品分析结论必须同时包含「存量盘活洞察」（如同区域竞品存量客户激活手段、我方存量客户流失原因）和「增量增长机会」（如增量竞品未覆盖客群、新市场进入机会）

## 竞品分析维度（酒店行业专用）

### 1. 产品维度

```
- 客房面积与配置（平米数、床型、景观）
- 餐饮（早餐/中餐/下午茶/酒吧）
- 会议设施（规模、配置）
- 康乐设施（泳池、健身房、SPA）
- 特色服务（管家、欢迎礼、会员权益）
```

### 2. 定价维度

```
- 平日均价（OTA价格）
- 周末/节假日价格浮动
- 套餐打包价（如婚宴、会议套餐）
- 会员价格体系
```

### 3. 渠道维度

```
- OTA占比（携程/美团/飞猪/抖音）
- 官网/APP直销比例
- 企业协议客户占比
- 会员体系（积分、权益、等级）
```

### 4. 运营维度

```
- OCC（出租率）水平
- RevPAR（每可售房收入）
- 评分（携程/美团/飞猪）
- 差评关键词
```

## 分析框架

### Step 1：确认对标对象

根据酒店定位和区域，选择2-3个核心竞品：

```
竞品选择原则：
- 直接竞品（同定位、同区域）
- 标杆竞品（行业标杆、学习对象）
- 替代竞品（可能抢走客户的其他选择）
```

### Step 2：数据收集

| 数据类型 | 获取渠道 |
|----------|----------|
| 价格 | OTA官网、携程、美团 |
| 产品配置 | 各品牌官网、酒店详情页 |
| 评分/口碑 | 携程、美团、飞猪 |
| 会员体系 | 各品牌会员页面 |
| 位置/交通 | 地图+实际体验 |

## 容错机制

### 异常处理

- **竞品数据无法获取（OTA页面访问失败/被反爬）**：切换数据源；携程失败→尝试美团/飞猪；均失败→使用公开新闻/财报数据+标注「⚠️ 部分数据来自公开资料，建议线下核实」
- **竞品选择不明确（用户未指定具体对手）**：自动选择第一梯队（开元、华住、锦江、亚朵）；标注「已默认选择直接竞品，如需调整请告知」
- **酒店行业专用维度数据缺失（如OCC/RevPAR等内部数据）**：使用公开可得的评分/价格/设施信息替代；标注「⚠️ [维度]数据为行业估算值，非实际运营数据」
- **虫洞AI能力对比信息过时**：标注「⚠️ 虫洞AI功能信息基于[日期]前公开资料，建议确认最新功能清单」
- **分析报告输出过长**：自动压缩为核心发现（3点优势+3点机会+3点威胁），详细对比表格折叠为附录

### 决策边界

| 情况 | 决策 |
|------|------|
| 竞品数量 < 2 | 至少选择2个竞品（含1个直接竞品+1个区域竞品）方可输出 |
| 数据可信度低（多源冲突） | 列出所有来源，标注置信度；不强制选"正确"数据 |
| 用户要求与德胧定位不符的分析 | 提醒并建议调整范围，但不拒绝执行 |
| 532会员战略数据缺失 | 标注「李和聪532模型数据待补充」，使用通用酒店会员逻辑替代 |

### 检查点机制

每完成一个分析维度输出状态快照：`📍 维度[X/5] 完成 | 产品配置：✓ | 定价：✓ | 渠道：⚠️待补充 | ...`
