四、经验总结与最佳实践

✅ 成功经验

1. 纯静态 HTML 原型优先

用 HTML+CSS+JS 快速验证 AI 产品想法，零依赖、零构建、30分钟出原型，是 AI 产品探索阶段的最优解。

2. 真实 API 优先于模拟数据

集成高德 Web 服务 API 替代模拟数据是关键转折点。真实数据极大提升演示可信度，地图标注从"大概位置"变成精确坐标。

3. 双路 API 策略（JSAPI + Web 服务）

高德 JSAPI 负责前端地图渲染，Web 服务 API 负责后端数据查询，两者互补，形成完整闭环。

4. SKILL 复用机制价值高

WorkBuddy SKILL 体系让能力沉淀可被复用。一次配置，后续对话直接"加载XX技能"即可激活对应能力。

❌ 踩过的坑

1. 模拟地理编码导致坐标偏移

问题：硬编码坐标模拟地理编码，导致天津酒店定位跑到北京通州。解决：改用高德 Web 服务 API。

2. AMap 安全密钥（Security Js Code）缺失

前端 securityJsCode 未配置，某些功能无法使用。现状：待在高德控制台获取并填入。

3. 服务器端口冲突

8080 端口已被占用，临时切换到 8088/8900。建议统一规划端口分配。

4. 钉钉企业应用消息权限

AccessToken 获取成功，但无法发消息（缺权限）。需在钉钉开放平台申请消息发送权限并重新审批。
